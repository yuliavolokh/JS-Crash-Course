Git Clone Command

Description: The "git clone" command is used to create a copy of a Git repository on your local machine.

Syntax:

git clone <repository URL>

Examples:

  • git clone https://github.com/user/repo.git
  • git clone git@github.com:user/repo.git

Explanation: The "git clone" command takes a repository URL as its argument and creates a copy of the repository on your local machine. The URL can be a HTTPS URL or a SSH URL. Once the clone is complete, you can make changes to the code and push them back to the original repository.

Options:

Option Description
--depth=n Creates a shallow clone with a history truncated to the specified number of commits
--branch=name Clone only a specific branch instead of the entire repository
--recursive Clones submodules recursively

Git Push Command

Description: The "git push" command is used to upload local repository content to a remote repository.

Syntax:

git push <remote> <branch>

Examples:

  • git push origin master
  • git push heroku main

Explanation: The "git push" command takes two arguments - the remote repository and the branch you want to push. The remote can be a name (e.g. origin) or a URL. The branch can be a name (e.g. master) or a reference (e.g. HEAD). Once the push is complete, the changes will be reflected in the remote repository.

Options:

Option Description
-f, --force Forces the push, even if it results in a non-fast-forward merge
--tags Pushes all tags to the remote repository
--dry-run Simulates the push without actually pushing any changes

Git Revert Command

Description: The "git revert" command is used to create a new commit that undoes the changes made by a previous commit.

Syntax:

git revert <commit>

Examples:

  • git revert HEAD
  • git revert a1b2c3d4

Explanation: The "git revert" command takes a commit as its argument and creates a new commit that undoes the changes made by that commit. This is done by creating a new commit that applies the inverse of the changes made in the original commit. The new commit will have the opposite effect of the original commit.
